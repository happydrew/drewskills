# 执行规则

## 暴涨词筛选规则

- 使用 `Related queries > Rising`
- 只保留 `> 2000%`、`Breakout`，或等价的本地化暴涨标签，例如 `爆发`、`飙升`

## 候选词通过规则

1. 关键词必须是在最近 7 天内新出现的词
2. 从第一次出现非零搜索量之后开始看，其平均水平应接近或达到 `GPTs`
3. 如果尾部又回落到 `0` 或接近 `0`，则判定为过短期词，不通过

## 稳定执行规则

- 必须串行执行，不能并行
- 在词根组之间、compare 请求之间都要等待
- 如果 `relatedsearches` 失败，先按健康问题处理，不要先当作“没有数据”
- 第一次失败后，等待 `60-90` 秒再重试
- 第二次失败后，等待 `3-5` 分钟再重试
- 两次重试后仍失败，则把该项标记为 `deferred`

## 写回规则

- 写入前先读取目标 Feishu 区域
- 只写入新增行
- 写入后等待 `3-5` 秒，再读回对应区域
- 如果第一次回读还是旧数据，再多等一会并只重试一次
